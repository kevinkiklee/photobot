2. Internal Bot Auth (JWT & RLS)

Secure Data Access

How the Bot and Dashboard talk to Supabase safely.

  • Service Role Key: The Bot (on Railway) uses a protected server-side key to bypass row-level security for high-speed lookups.
  • Row Level Security (RLS): The Dashboard API uses user-specific JWTs to ensure Admins can only see settings for their own servers.

Security Benefits

  • No Passwords: You never store or manage user passwords—Discord handles all credentials and MFA.
  • Instant Revocation: If someone is kicked from the Discord server or loses their Admin role, their Dashboard access is instantly revoked.

Implementation Detail

  • Session Management: Supabase handles the JWT rotation and session persistence automatically in the browser.
