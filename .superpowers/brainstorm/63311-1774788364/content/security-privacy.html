Security & Privacy: The "Fortress" Model

Protecting user copyright and server integrity at scale.

1. Image Privacy & Zero-Retention

Volatile-Only Processing

Images for /palette or /critique are never written to disk.

Buffer input = await fetch(url);
let result = process(input);
input = null; // Purge from RAM immediately

✓ Zero-training guarantee: APIs are configured to not use user uploads for model training.

2. Admin Safety & Scoping

Secure /settings Execution

Preventing "Admin Account Hijacking" from wrecking the server.

  • MFA Requirement: Optional toggle requiring Discord 2FA for anyone running /settings.
  • Sensitive Command Scoping: Features like "Full Data Export" require owner-level ID verification.

Data Protection

  • EXIF Stripping: Automatically strip GPS/Personal data from images before AI analysis.
  • GDPR/CCPA: Provide a /my-data delete command for users to purge their critique history.

System Security

  • Token Safety: Use AWS Secrets Manager or Environment Variables (never hardcoded).
  • Rate Limiting: Per-user and Per-IP limits to prevent "DDoS-by-API-Calls" which could spike costs.
