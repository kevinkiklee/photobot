Hybrid Permissions: Native Discord + Custom PhotoBot

Best of both worlds: Broad visibility + Granular control.

Native Discord: Visibility Control

Command Permissions (Discord UI)

Admins can hide commands entirely from certain roles.

/critique:
✓ Role: @Professional
✗ Role: @Everyone

Result: Only @Professional users see /critique in their command list.

PhotoBot: Channel & Specific Overrides

Internal Settings (/settings)

Fine-grained control that Discord's UI lacks.

$ /settings feature:critique channel:#general state:disable

Result: Even @Professional users are blocked if they try /critique in #general.

Why this is the way

  • Less noise: Users only see the commands they're allowed to use.
  • Power: Admins can still prevent bot-spam in specific channels without disabling the command globally.
  • Efficiency: Discord's native check happens before our bot even receives the command, saving resources.

The Catch

  • Double Config: Admins need to understand that Discord controls *visibility* and PhotoBot controls *execution*.
  • Guidance: We should add a "Sync Check" to `/settings list` that warns if Discord permissions contradict our internal settings.
