Approaches to Feature Toggling

Choose the structure that best fits your server management needs.

A

Hierarchical Inheritance

Global Server Setting → Channel Overrides → Role Permissions.

  • Simple to understand: "Turn it on for the server, then off for specific channels."
  • Role permissions act as the final gate: "Who can actually use it?"
B

Configuration Matrix

A single grid view of Features × (Servers, Channels, Roles).

  • Total visibility: See every permission at once in the dashboard.
  • More complex to manage via Discord commands, but powerful for admins.
C

Permission Tags

Features are treated like Discord Permissions.

  • Integrates directly with Discord's native permission system (Manage Channels, etc.).
  • Admins use `/allow [feature] [role/channel]` to build a "whitelist."
Example: Hierarchical View
Dashboard | Features | Permissions
Feature: /critique [Enabled ✓]
Channel Overrides:
#portfolio-review: Enabled
#general: Disabled
Role Overrides:
Professional: Allowed
Beginner: Allowed (Cooldown active)
