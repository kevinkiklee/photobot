Discord: Command Interface

@Admin /settings feature:critique state:disable channel:#general

> Feature /critique is now disabled for channel #general.


@Admin /settings feature:critique allow-role:@Professional

> Feature /critique is now restricted to @Professional and higher.

Why it works

  • Safe by default: Features are either globally ON or OFF.
  • Specific: One command fixes one problem (e.g., "stop bot spam in #general").
  • Scalable: New servers get the default setup without configuration.

Trade-offs

  • Potential conflicts: If a user has two roles (one allowed, one disallowed), we need a clear priority rule.
  • Discovery: Admins need a way to see all active overrides (a `/settings list` command).
