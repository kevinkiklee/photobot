PhotoBot-First Permission Management

A single source of truth for all bot features.

The PhotoBot Command Suite

One Command, Full Control

Admins manage everything via /settings.

# Server-wide
$ /settings feature:trivia state:enable

# Role-specific (PhotoBot Internal)
$ /settings feature:critique role:@Pros state:allow

# Channel-specific
$ /settings feature:critique channel:#general state:disable
Why This Approach?

The "Single Source of Truth"

Instead of jumping between Discord's UI and the bot, admins stay in one workflow.

  • Consistent: All features follow the same logic.
  • Portable: Settings are stored in our DB, making it easier to migrate or backup.
  • Audit-Ready: Every change goes through our config_audit_logs table.

The "Hidden" Benefit

By keeping role-based access inside PhotoBot, we can implement custom logic that Discord doesn't support, like:

  • Cooldowns by Role: @Pros get unlimited /critique, @Newbies get 1 per day.
  • Feature Bundles: "Enable all 'Visual Utility' features for @Mentors."
