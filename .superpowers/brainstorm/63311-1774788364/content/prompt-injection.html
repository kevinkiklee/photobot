Defeating Prompt Injection: The "Sanitized Vision" Strategy

Ensuring user-submitted intent and image metadata don't hijack the bot's instructions.

1. Structural Isolation (Input Sandboxing)

The "System Message" Fortress

Strictly separating user input from core instructions.

SYSTEM: You are a photography mentor.
USER: [IMAGE] + [SANITIZED_TEXT]

Even if USER says "Forget all instructions," the SYSTEM context remains the primary truth for the model.

2. The "Pre-Flight" Guard

Keyword & Pattern Filtering

Blocking known injection techniques before they reach the AI.

  • Injection Signature Check: Flagging strings like "Ignore previous instructions" or "system override."
  • Delimited Input: Wrapping user text in unique tokens (e.g., [USER_INPUT] ... [/USER_INPUT]) to help the model distinguish content from commands.

Active Defense

  • Vision-Only Mode: For commands like /palette, the bot only looks at the pixels, completely ignoring any text the user might have hidden in the image metadata.
  • Output Validation: If the AI response contains unexpected keywords (e.g., "Developer Mode" or "Internal Config"), the response is blocked.

Metadata Hardening

  • EXIF Stripping: Malicious users can hide injection payloads in "Camera Model" or "Artist" EXIF tags. We purge all metadata before analysis.
  • Token Limiting: Restricting user "intent" tags to a strict character limit (e.g., 200 chars) to prevent massive payloads.
